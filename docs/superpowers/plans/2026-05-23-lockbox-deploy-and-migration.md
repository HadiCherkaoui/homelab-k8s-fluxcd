# Lockbox Deploy + SOPS Migration Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Deploy `lockbox` server and `lockbox-k8s-controller` via FluxCD into `lockbox-system` namespace, then migrate all 18 existing SOPS-encrypted secrets into lockbox so the controller becomes the runtime source of `core/v1.Secret` objects (the `secrets/lockbox/*` bootstrap material stays SOPS-managed).

**Architecture:** Two OCI HelmReleases under `infrastructure/lockbox/`, sharing one namespace. The `lbx` CLI (already installed at `/usr/local/bin/lbx`) and the controller share one Ed25519 keypair — because lockbox uses the raw Ed25519 seed as the AES-256-GCM key (`crates/lockbox-crypto/src/cipher.rs:13`), different keypairs cannot decrypt each other's ciphertext. The 32-byte seed is generated by `lbx init`, then SOPS-encrypted as `lockbox-credentials` Secret so the controller loads the same seed at boot.

**Tech Stack:** FluxCD v2.6, Kustomize, SOPS+age, Helm (OCI charts), Traefik IngressRoute, lockbox v0.1.0 chart `0.1.0-0df3cdfc`, lockbox-k8s-controller chart `0.1.0`.

**Spec reference:** `docs/superpowers/specs/2026-05-23-lockbox-deploy-and-migration-design.md`.

- DNS for `lockbox.cherkaoui.ch` must already point at the cluster's Traefik LB IP. - `lbx` CLI is already on PATH (`/usr/local/bin/lbx`, version 0.1.0). Existing state at `~/.local/share/lockbox/` points at `http://localhost:8080` from local dev — Task 12 wipes it.
- All commits should follow the repo's commit-message style. End every message with `Co-Authored-By: Claude Opus 4.7 (1M context) <noreply@anthropic.com>`.

## Phase D — Migrate existing SOPS secrets to lockbox

### Task 15: Write the migration helper script

**Files:**
- Create: `scripts/push-sops-to-lockbox.sh`

- [ ] **Step 1: Write the script**

```bash
cat > scripts/push-sops-to-lockbox.sh <<'BASH'
#!/usr/bin/env bash
# push-sops-to-lockbox.sh — one-shot migration helper.
#
# Iterates every secrets/**/*.secret.yaml EXCEPT secrets/lockbox/* (which is
# bootstrap material that stays SOPS-managed). For each file:
#   1. sops --decrypt the file
#   2. yq-extract metadata.name + metadata.namespace + stringData + data
#   3. base64-decode any `data:` values back to strings (lbx wants string KV pairs)
#   4. lbx set -n <namespace> <name> KEY1=value1 KEY2=value2 ...
#
# Idempotent: lbx set overwrites, so re-running is safe.
#
# Requires: sops, yq (mikefarah v4+), lbx; SOPS_AGE_KEY exported.
set -euo pipefail

if [[ -z "${SOPS_AGE_KEY:-}" ]]; then
  echo "ERROR: SOPS_AGE_KEY must be exported" >&2
  exit 1
fi

ROOT="$(cd "$(dirname "$0")/.." && pwd)"
cd "$ROOT"

FAILED=()
PUSHED=()

while IFS= read -r -d '' f; do
  case "$f" in
    secrets/lockbox/*) continue ;;
  esac

  plain=$(sops --decrypt "$f")
  name=$(echo "$plain" | yq -r '.metadata.name')
  ns=$(echo "$plain" | yq -r '.metadata.namespace')
  if [[ -z "$name" || -z "$ns" || "$name" == "null" || "$ns" == "null" ]]; then
    echo "  skip (no name/namespace): $f" >&2
    FAILED+=("$f")
    continue
  fi

  pairs=()
  # stringData values are plain strings
  while IFS=$'\t' read -r k v; do
    [[ -z "$k" ]] && continue
    pairs+=("${k}=${v}")
  done < <(echo "$plain" | yq -r '.stringData // {} | to_entries | .[] | [.key, .value] | @tsv')

  # data values are base64
  while IFS=$'\t' read -r k v; do
    [[ -z "$k" ]] && continue
    decoded=$(printf '%s' "$v" | base64 -d)
    pairs+=("${k}=${decoded}")
  done < <(echo "$plain" | yq -r '.data // {} | to_entries | .[] | [.key, .value] | @tsv')

  if [[ ${#pairs[@]} -eq 0 ]]; then
    echo "  skip (no data): $f" >&2
    FAILED+=("$f")
    continue
  fi

  echo "push: $f → namespace=$ns name=$name (${#pairs[@]} keys)"
  if lbx set -n "$ns" "$name" "${pairs[@]}" >/dev/null; then
    PUSHED+=("$ns/$name")
  else
    echo "  lbx set FAILED for $ns/$name" >&2
    FAILED+=("$f")
  fi
done < <(find secrets -type f -name '*.secret.yaml' -print0)

echo
echo "Pushed (${#PUSHED[@]}):"
printf '  - %s\n' "${PUSHED[@]}"
if [[ ${#FAILED[@]} -gt 0 ]]; then
  echo
  echo "FAILED (${#FAILED[@]}):"
  printf '  - %s\n' "${FAILED[@]}"
  exit 1
fi
BASH
chmod +x scripts/push-sops-to-lockbox.sh
```
